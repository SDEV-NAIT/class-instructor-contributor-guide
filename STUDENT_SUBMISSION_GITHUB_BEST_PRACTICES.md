# Student Submission GitHub Best Practices

Submission guidelines that instructors need to clearly communicate to students.

## Incremental Development via Commit History - Requiring frequent, meaningful commits throughout development, rather than relying on a single final upload. Reviewing the commit history alongside the final code provides much stronger evidence of authentic progress and encourages better long-term development habits.

Please include the following text in your assignments:

You must demonstrate incremental development of your solution. This means that you must begin work on your solution as soon as possible and commit often to the assignment repository. Each commit must demonstrate functional improvements to the solution. Just like in a professional environment, leaving a task untouched until right before the deadline is not acceptable — start early and make consistent progress. Failure to show incremental work during the assignment period may result in loss of marks.

_IMPORTANT NOTE_ This is going to be on a per course basis
- Course leads may provide more details on how to apply this.
- On brightspace this course leaders may instruct to on how to apply this whether it's a deduction or part of a rubric.

## If a student submits code that differs greatly from what was demonstrated in class it must be documented (e.g. comments, citations, etc.) and students may be asked to provide a verbal explanation of how the code works to your instructor. Failure to explain any code that the student submitted will be considered as potential evidence of academic misconduct and may trigger an investigation, potentially resulting in further consequences.
- Please add these expectations to your assignment and clearly communicate this to students.
- Course leaders/courses may have stricter requirements than this but this is the minimum expectation for all courses.

## Github/Brightspace Submission Standards - Students should be expected to submit a link to their repository on brightspace in addition to pushing all their code to github

- This is to encourage students to use github for their submissions and to encourage them to submit on time.
- If they forget to submit the link on brightspace but have all of the commits before the deadline this may be dealt with on a dcae by case basis.

**IMPORTANT NOTE** If the course requires a submission process where the students submit a video, other documents or any other requirements as per the assignment instructions and they do not submit it to brightspace (as instructed in the assignment/lab/quiz/etc.) then they may be penalized accordingly.

## (If Instructed) students are expected to provide clear project documentation.

- Requiring a detailed README.md that includes the project purpose, setup instructions, required screenshots/demo evidence, and usage examples. This improves student communication skills and makes submissions much easier to evaluate and troubleshoot.
